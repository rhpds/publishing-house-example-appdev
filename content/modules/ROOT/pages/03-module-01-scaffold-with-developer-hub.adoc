You want a running application on OpenShift. Not tomorrow. Not after writing YAML. Right now.

Red Hat Developer Hub makes that possible. It provides golden path templates, pre-built starting points that include application code, CI/CD pipelines, deployment manifests, and GitOps configuration. You pick a template, fill in a short form, and the platform does the rest.

In this lab, you will scaffold the Postcard Generator from a golden path template and have it running on OpenShift in minutes.

== Learning objectives

By the end of this lab, you will be able to:

* Navigate the Red Hat Developer Hub software catalog and template gallery
* Scaffold a new application from a golden path template
* Verify that the scaffolded app is deployed and running on OpenShift
* Understand how platform teams use templates to encode best practices

== Exercise 1: Explore Developer Hub

Developer Hub is where your organization's platform team publishes golden path templates. Think of it as an app store for internal applications, but instead of downloading apps, you scaffold new ones from proven patterns.

. Open Red Hat Developer Hub in your browser:
+
{devhub_url}

. Sign in by clicking *Log in with OpenShift* and using your lab credentials:
+
* Username: `{user}`
* Password: `{password}`

. After signing in, you land on the Developer Hub home page. Click *Catalog* in the left navigation.
+
The Software Catalog shows components already registered on this platform. Each entry represents an application, service, or resource that the platform team has onboarded. Browse the list to get a feel for how organizations inventory their software.

. Now click *Create* in the left navigation.
+
This is the template gallery. Each card represents a golden path template, a pre-configured starting point for a specific type of application. Templates are created by platform teams and consumed by developers.

. Find the *Postcard Generator* template card. Notice the description, tags, and owner information.
+
This template will create everything you need: a Git repository with the application source code, a Tekton pipeline for CI/CD, Kubernetes deployment manifests, and an Argo CD application for GitOps.

== Exercise 2: Scaffold the Postcard Generator

Time to create your app. The template handles everything, you just fill in a few fields.

. Click *Choose* on the Postcard Generator template.

. Fill in the scaffolding form:
+
* *Project name*: `postcard-generator`
* *Owner*: Select your user from the dropdown
* *Namespace*: Your pre-assigned OpenShift namespace (shown in the dropdown)

. Click *Next* to review the summary.
+
Take a moment to read what will be created:
+
* A Git repository in Gitea with the full application source
* A Tekton Pipeline for building and deploying the app
* Kubernetes manifests (Deployment, Service, Route)
* An Argo CD Application for GitOps management

. Click *Create* and watch the progress.
+
Each step executes in sequence:

* Repository created in Gitea
* Source code and CI/CD configuration committed
* OpenShift resources applied
* Application deployment triggered
+
This takes about 60 seconds.

. Once all steps show green checkmarks, click *Open Component* to view your new app in the Developer Hub catalog.

== Exercise 3: Tour your new app

The scaffolding created your app and deployed it to OpenShift. Let's see it running.

. From the component page in Developer Hub, look for the *OpenShift* or *Topology* link. Click it to open the OpenShift developer topology view in a new tab.
+
Alternatively, open the OpenShift Console directly:
+
{openshift_console_url}
+
Sign in with `{user}` / `{password}`, switch to the *Developer* perspective, and select your namespace from the project dropdown.

. In the Topology view, you should see the Postcard Generator deployment represented as a node.
+
Identify the key resources:
+
* *Deployment* -- the running application
* *Pod* -- the container instance
* *Service* -- internal networking
* *Route* -- the external URL

. Click the *Open URL* icon (the small arrow in the top-right corner of the deployment node) to open the Postcard Generator in a new browser tab.
+
Alternatively, navigate directly to:
+
{postcard_app_url}

. You should see the Postcard Generator interface. Enter your name in the *From* field.

. Click *Generate Postcard*.
+
A random destination is selected and a retro-styled postcard appears with:
+
* A colorful travel-poster design
* The destination name and a fun tagline
* An interactive map pinning the location
* Your name as the sender

. Click *Generate Postcard* a few more times. Each postcard picks a different destination with its own color scheme and tagline.

Run the following in your terminal to confirm the deployment is healthy:

[source,role="execute"]

oc get deployment postcard-generator -n {user}-postcard-generator
----

----
NAME                  READY   UP-TO-DATE   AVAILABLE   AGE
postcard-generator    1/1     1            1           5m
----

* READY shows 1/1
* The Postcard Generator UI loads in your browser
* You successfully generated at least one postcard
