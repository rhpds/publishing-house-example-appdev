You have a running Postcard Generator on OpenShift. Now you want to change it. Add a destination. Tweak the colors. Make it yours.

OpenShift Dev Spaces gives you a cloud-hosted development environment that runs directly on the cluster. No local setup. No "works on my machine." You open a browser, start coding, and see your changes reflected instantly.

This lab is not about the IDE. It is about how fast and close the integration is between your code and the running platform.

== Learning objectives

By the end of this lab, you will be able to:

* Launch a Dev Spaces workspace from a Git repository
* Modify application code and see changes reflected in real time via live reload
* Add new data to a running application without rebuilding or redeploying
* Explain how devfiles define reproducible development environments

== Exercise 1: Launch your workspace

Dev Spaces creates a full development environment from a devfile checked into your Git repository. The devfile defines the runtime, tools, and startup commands, making the environment as reproducible as a container image.

. Open OpenShift Dev Spaces in your browser:
+
{devspaces_url}

. Sign in by clicking *Log in with OpenShift* and using your lab credentials:
+
* Username: `{user}`
* Password: `{password}`

. On the Dev Spaces dashboard, paste your Postcard Generator Git repository URL in the *Import from Git* field:
+
[source,role="execute"]
----
{gitea_url}/{gitea_user}/postcard-generator.git
----

. Click *Create & Open*.
+
Dev Spaces reads the `devfile.yaml` from the repository and builds your workspace. This includes the Python runtime, dependencies, and port configuration. The workspace is ready in about 60 seconds.

. Once the IDE loads, open a terminal: click *Terminal* in the top menu, then *New Terminal*.

. Start the application in development mode:
+
[source,role="execute"]
----
cd /projects/postcard-generator && flask run --debug --host=0.0.0.0
----

. When a notification appears asking about port forwarding, click *Open in New Tab* (or *Yes* to redirect).
+
The Postcard Generator opens in a new browser tab, running directly from your Dev Spaces workspace.

. Generate a postcard to confirm the app is working.

== Exercise 2: Customize a postcard

The inner loop is where developers spend most of their time: change code, see the result, repeat. With Flask in debug mode, you do not need to rebuild or restart anything. Save the file, refresh the browser.

. In the Dev Spaces file explorer (left panel), navigate to:
+
`static/css/postcards.css`

. Find the `.postcard-frame` class. This controls the retro border and shadow effect around each postcard.

. Change the `border-color` value to something bold. For example:
+
[source,css]
----
border-color: #e74c3c;
----

. Save the file (*Ctrl+S* or *Cmd+S*). Do *not* restart the server.

. Switch to the browser tab with the Postcard Generator and refresh the page.
+
The postcard frame now displays your new border color. No rebuild. No container image. No deployment. Just save and refresh.

. Next, open:
+
`templates/postcard.html`

. Find the tagline element in the template. Change the formatting, for example, add a travel emoji or adjust the phrasing style.

. Save the file and refresh the browser tab again.
+
Your change appears immediately.

This is the inner loop: *code, save, refresh*. The development environment is running on the same cluster as the application, and live reload means your feedback loop is seconds, not minutes.

== Exercise 3: Add a new destination

The Postcard Generator pulls destinations from a curated JSON file. Adding a new destination is as simple as editing that file.

. In the Dev Spaces file explorer, open:
+
`data/destinations.json`

. Review the structure of an existing destination entry. Each one has a name, country, coordinates, tagline, and color scheme.

. Add a new destination at the end of the list. Pick somewhere fun:
+
[source,json]
----
{
  "name": "Reykjavik",
  "country": "Iceland",
  "lat": 64.1466,
  "lng": -21.9426,
  "tagline": "Greetings from Reykjavik -- Land of Fire and Ice!",
  "colors": {
    "primary": "#1a5276",
    "accent": "#48c9b0"
  }
}
----
+
NOTE: Make sure to add a comma after the previous entry in the JSON array.

. Save the file.

. In the Postcard Generator browser tab, click *Generate Postcard* repeatedly until your new destination appears.
+
Your destination shows up with the coordinates pinned on the map, your tagline, and your chosen color scheme.
